# Seat 座位编排系统 - 响应式设计规范

## 1. 概述

本文档定义了 Seat 座位编排系统的响应式设计规范，确保应用在各种设备（手机、平板、桌面端）上都能提供最佳的用户体验。

## 2. 断点系统

### 2.1 断点定义

| 断点名称 | 最小宽度 | 最大宽度 | 目标设备          |
| -------- | -------- | -------- | ----------------- |
| `xs`     | 0px      | 575px    | 小型手机          |
| `sm`     | 576px    | 767px    | 大型手机          |
| `md`     | 768px    | 991px    | 平板竖屏          |
| `lg`     | 992px    | 1199px   | 平板横屏/小型桌面 |
| `xl`     | 1200px   | 1399px   | 标准桌面          |
| `xxl`    | 1400px   | 无限     | 大型桌面          |

### 2.2 CSS 变量

```css
:root {
  --breakpoint-xs: 0;
  --breakpoint-sm: 576px;
  --breakpoint-md: 768px;
  --breakpoint-lg: 992px;
  --breakpoint-xl: 1200px;
  --breakpoint-xxl: 1400px;
}
```

## 4. 组件响应式规范

### 4.1 座位网格 (SeatGrid)

| 屏幕尺寸 | 座位单元格尺寸 | 缩放方式     | 滚动方式        |
| -------- | -------------- | ------------ | --------------- |
| 桌面端   | 80x60px        | 无缩放       | 容器滚动        |
| 平板     | 70x52px        | 自适应缩放   | 双指缩放 + 滚动 |
| 手机     | 60x45px        | 默认缩放 80% | 双指缩放 + 滚动 |

### 4.2 工具栏 (Toolbar)

| 屏幕尺寸 | 布局方式            | 按钮尺寸     |
| -------- | ------------------- | ------------ |
| 桌面端   | 水平排列，左右分布  | 14px padding |
| 平板     | 水平排列，居中      | 12px padding |
| 手机     | 折叠菜单 + 核心按钮 | 10px padding |

### 4.3 侧边栏

| 屏幕尺寸 | 显示方式     | 宽度         |
| -------- | ------------ | ------------ |
| 桌面端   | 固定显示     | 320px        |
| 平板     | 单侧固定显示 | 280px        |
| 手机     | 抽屉组件     | 85% 屏幕宽度 |

## 5. 触摸交互规范

### 5.1 触摸目标尺寸

- 最小触摸目标：44x44px
- 推荐触摸目标：48x48px
- 触摸目标间距：8px

### 5.2 手势支持

| 手势     | 功能           |
| -------- | -------------- |
| 单击     | 选择座位/学生  |
| 长按     | 显示上下文菜单 |
| 拖拽     | 移动学生到座位 |
| 双指缩放 | 缩放座位网格   |
| 双指滑动 | 平移座位网格   |

### 5.3 触摸反馈

- 按下状态：背景色变化 + 轻微缩放 (scale: 0.98)
- 拖拽状态：透明度降低 + 阴影增强
- 放置目标：边框高亮 + 脉冲动画

## 8. 动画与过渡

### 8.1 过渡时间

- 快速：150ms
- 标准：250ms
- 慢速：350ms

### 8.2 缓动函数

```css
--ease-out: cubic-bezier(0.4, 0, 0.2, 1);
--ease-in: cubic-bezier(0.4, 0, 1, 1);
--ease-in-out: cubic-bezier(0.4, 0, 0.2, 1);
```

### 8.3 移动端优化

- 禁用复杂的悬停动画
- 使用 `transform` 和 `opacity` 实现动画（GPU 加速）
- 减少动画数量以提升性能

## 9. 性能优化

### 9.1 图片优化

- 使用 CSS 渐变替代装饰性图片
- 图标使用 SVG 格式
- 懒加载非关键图片

### 9.2 渲染优化

- 使用 `will-change` 提示浏览器优化
- 避免强制同步布局
- 使用虚拟滚动处理长列表

### 9.3 移动端优化

- 减少重绘和回流
- 使用 `touch-action` 控制触摸行为
- 禁用不必要的阴影和模糊效果
